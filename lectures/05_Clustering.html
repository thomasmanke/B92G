Clustering Approaches

Hierarchical Clustering

Hierarchical clustering requires:

  • a distance definition, for example Euclidean distance
  • an agglomeration method
    • complete linkage: cluster distance is maximal distance between cluster members
    • average linkage: cluster distance is average distance between cluster members
  • optional: cut prescription, that is, at which height to cut the tree
TipMore Agglomeration Methods

Assume that distance between all elements are defined \(d(a,b)\) and initially each elements defines its own cluster.

Clusters are merged iteratively - nearest clusters first: \(C = A\cup B\).

New distances \(D(C,X)\) between merged clusters \(C\) and old cluster \(X\) can be defined from old distances by various possible iterative updates. For example:

Method Update rule (\(C=A\cup B\))
Single linkage \(D(C,X)=\min\!\left(D(A,X),\,D(B,X)\right)\)
Complete linkage \(D(C,X)=\max\!\left(D(A,X),\,D(B,X)\right)\)
WPGMA \(D(C,X)=\displaystyle\frac{D(A,X)+D(B,X)}{2}\)
UPGMA \(D(C,X)=\displaystyle\frac{|A|\,D(A,X)+|B|\,D(B,X)}{|A|+|B|}\)

Reference: Wikipedia Comparisons.

ImportantHierarchical Clustering is very expensive

For \(N\) samples:

  • Space: \(O(N^2)\) (distance matrix)
  • Time (naive): \(O(N^2)\) (distances) \(\times O(N)\) (merge steps)
  • Time (efficient): \(O(N^3) \to O(N^2)\) avoid repeated searches over all cluster pairs Sibson (1973), SLINK

k-means implementation 
def run_kmeans_history(data, k=2, max_iter=10, random_state=123):
    """
    Run a simple k-means loop and keep track of the iterations
    This is slow than the bona-fide KMeans which does not keep track of iterations

    Parameters
    ----------
    data : numpy array, shape (n_samples, n_features)
        Numerical feature matrix.
    k : int
        Number of clusters.
    max_iter : int
        Maximum number of k-means iterations.
    random_state : int
        Random seed.

    Returns
    -------
    points : DataFrame
        Original data repeated for every iteration, with cluster labels.
    centers : DataFrame
        Centroid coordinates for every iteration.
    """
    rng = np.random.default_rng(random_state)
    X = np.asarray(data)

    n_samples, n_features = X.shape

    feature_names = [f"dim_{i}" for i in range(n_features)]

    centers = rng.uniform(
        low=X.min(axis=0),
        high=X.max(axis=0),
        size=(k, n_features),
    )

    point_frames = []
    center_frames = []

    for iteration in range(1, max_iter + 1):
        # E-step: assign samples to nearest centroid
        distances = ((X[:, None, :] - centers[None, :, :]) ** 2).sum(axis=2)
        labels = distances.argmin(axis=1)

        points = pd.DataFrame(X, columns=feature_names)
        points["sample_id"] = np.arange(n_samples)
        points["cluster"] = labels.astype(str)
        points["iteration"] = iteration
        point_frames.append(points)

        centroids = pd.DataFrame(centers, columns=feature_names)
        centroids["sample_id"] = "centroid_" + pd.Series(range(k)).astype(str)
        centroids["cluster"] = [str(i) for i in range(k)]
        centroids["iteration"] = iteration
        center_frames.append(centroids)

        # M-step: update centroid positions
        new_centers = centers.copy()

        for j in range(k):
            if np.any(labels == j):
                new_centers[j] = X[labels == j].mean(axis=0)

        if np.allclose(new_centers, centers):
            break

        centers = new_centers

    return (
        pd.concat(point_frames, ignore_index=True),
        pd.concat(center_frames, ignore_index=True),
    )

ImportantComputational Cost

\(N\) observations, \(D\) dimensions, \(K\) clusters

  • Space: \(N\) observations + \(K\) cluster centroids \(\to O(N+K)\)
  • Time: \(O(NKD)\) / iterations (find nearest centroid) \(\to O(N)\)
  • \(K\) (number of clsuters) and \(I\) (number of iterations) are usually small
NoteLoss Function and Some Maths

The Loss Function

\[L = \sum_{l=1}^k \sum_{i \in C_l} ||x_i - \mu_l||^2 = \sum_{l=1}^k \sum_{i=1}^n a_{il} ||x_i - \mu_l||^2 \to minimize\]

The loss is also called within-cluster sum of squares because it sums over all squared distances of samples from the centroid to which they belong. Unknown are the cluster assignments \(a_{il}\) and the centroids \(\mu_l\).

Algorithmic challenge: given sample vectors \(x_i\), determine cluster assignment \(a_{il} \in [0,1]\) and centroids \(\mu_l\).

Expectation-Maximization-like iteration

  1. Initialize model: \(\mu_l\)
  2. Iterate until convergence
    1. Assign each \(x_i\) to the cluster with nearest centroid: \(a_{il}\)
    2. Update centroid: \(\mu_j = 1/n_j \sum_i x_i\)

Method for initialization:

This technique speeds up convergence. The algorithm implemented is "greedy k-means++". In scikit-learn, the total within-cluster sum of squares is called inertia_. Each cluster has its own within-cluster sum of squares, but often we are interested in the total sum over all clusters: this is what k-means minimizes for a given \(k\).

Screening cluster solutions

Increasing \(k\) will decrease within-cluster SS, but increase model complexity. For \(k=n\), each point would be its own centroid and within-cluster SS would be 0. Usually we look for a practical compromise.

Numerically: Adjusted Rand Index

Different cluster algorithms can produce different partitions and different numbers of clusters. Moreover, the cluster labels are arbitrary. Therefore we need a better method to compare cluster solutions. A common solution is to focus on all pairs of samples and consider how often they end up in the same cluster or different clusters, regardless of arbitrary cluster labels.

The Rand Index (Rand 1971) for two partitions is defined as follows:

\[RI = \frac{a+d}{a+b+c+d} \in [0,1]\]

The Adjusted Rand Index (ARI) measures the similarity between two cluster solutions, accounting for chance groupings. Specifically, it accounts for an expected RI that would be observed for random cluster assignments and for a maximally possible RI.

It ranges from -1 to 1, with 1 indicating perfect agreement and 0 indicating similarity expected by chance.

\[ARI = \frac{A - E}{M - E} \in [-1, +1]\]

  • \(A\) = number of cluster agreements of sample pairs
  • \(M\) = maximal possible number of agreements
  • \(E\) = expected number of agreements by chance
Code 
ari_score = adjusted_rand_score(cluster_hclust, cluster_kmeans)

## Clustering Approaches

### Hierarchical Clustering

Hierarchical clustering requires:

- a distance definition, for example Euclidean distance
- an agglomeration method
  - **complete linkage**: cluster distance is maximal distance between cluster members
  - **average linkage**: cluster distance is average distance between cluster members
- optional: cut prescription, that is, at which height to cut the tree




::: {.callout-tip collapse=True}
### More Agglomeration Methods

Assume that distance between all elements are defined $d(a,b)$ and initially each elements defines its own cluster.

Clusters are merged iteratively - nearest clusters first: $C = A\cup B$. 

New distances $D(C,X)$ between `merged` clusters $C$ and `old` cluster $X$ can be defined from old distances by various possible iterative updates. For example:

| Method |  Update rule ($C=A\cup B$) |
|:-------|:------------------------------|
| Single linkage |  $D(C,X)=\min\!\left(D(A,X),\,D(B,X)\right)$ |
| Complete linkage |  $D(C,X)=\max\!\left(D(A,X),\,D(B,X)\right)$ |
| WPGMA | $D(C,X)=\displaystyle\frac{D(A,X)+D(B,X)}{2}$ |
| UPGMA | $D(C,X)=\displaystyle\frac{|A|\,D(A,X)+|B|\,D(B,X)}{|A|+|B|}$ |

Reference: [Wikipedia Comparisons.](https://en.wikipedia.org/wiki/Complete-linkage_clustering#Comparison_with_other_linkages){target="_blank" rel="noopener"}
:::

#### Tree cutting

The height at which two branches merge in the tree corresponds to the distance between the two clusters being merged, as defined by the chosen linkage criterion. 

Notice that the order of branches and nodes in these hierarchical trees ("dendrograms") is arbitrary: branches can be flipped or rotated.

::: {.callout-important}
### Hierarchical Clustering is very expensive

For $N$ samples:

  - Space: $O(N^2)$ (distance matrix)
  - Time (naive): $O(N^2)$ (distances) $\times O(N)$ (merge steps)
  - Time (efficient): $O(N^3) \to O(N^2)$ avoid repeated searches over all cluster pairs
[Sibson (1973), SLINK](https://academic.oup.com/comjnl/article-abstract/16/1/30/434805){target="_blank" rel="noopener"}
:::
  


### k-means Clustering


#### A visualization

Some helper functions:

```{python}
#| label: define_kmeans_history
#| code-fold: true
#| code-summary: k-means implementation
def run_kmeans_history(data, k=2, max_iter=10, random_state=123):
    """
    Run a simple k-means loop and keep track of the iterations
    This is slow than the bona-fide KMeans which does not keep track of iterations

    Parameters
    ----------
    data : numpy array, shape (n_samples, n_features)
        Numerical feature matrix.
    k : int
        Number of clusters.
    max_iter : int
        Maximum number of k-means iterations.
    random_state : int
        Random seed.

    Returns
    -------
    points : DataFrame
        Original data repeated for every iteration, with cluster labels.
    centers : DataFrame
        Centroid coordinates for every iteration.
    """
    rng = np.random.default_rng(random_state)
    X = np.asarray(data)

    n_samples, n_features = X.shape

    feature_names = [f"dim_{i}" for i in range(n_features)]

    centers = rng.uniform(
        low=X.min(axis=0),
        high=X.max(axis=0),
        size=(k, n_features),
    )

    point_frames = []
    center_frames = []

    for iteration in range(1, max_iter + 1):
        # E-step: assign samples to nearest centroid
        distances = ((X[:, None, :] - centers[None, :, :]) ** 2).sum(axis=2)
        labels = distances.argmin(axis=1)

        points = pd.DataFrame(X, columns=feature_names)
        points["sample_id"] = np.arange(n_samples)
        points["cluster"] = labels.astype(str)
        points["iteration"] = iteration
        point_frames.append(points)

        centroids = pd.DataFrame(centers, columns=feature_names)
        centroids["sample_id"] = "centroid_" + pd.Series(range(k)).astype(str)
        centroids["cluster"] = [str(i) for i in range(k)]
        centroids["iteration"] = iteration
        center_frames.append(centroids)

        # M-step: update centroid positions
        new_centers = centers.copy()

        for j in range(k):
            if np.any(labels == j):
                new_centers[j] = X[labels == j].mean(axis=0)

        if np.allclose(new_centers, centers):
            break

        centers = new_centers

    return (
        pd.concat(point_frames, ignore_index=True),
        pd.concat(center_frames, ignore_index=True),
    )
```

::: {.callout-important}
#### Computational Cost

$N$ observations, $D$ dimensions, $K$ clusters

- Space: $N$ observations + $K$ cluster centroids $\to O(N+K)$
- Time: $O(NKD)$ / iterations (find nearest centroid) $\to O(N)$
- $K$ (number of clsuters) and $I$ (number of iterations) are usually small
:::





::: {.callout-note collapse="true"}
#### Loss Function and Some Maths

The **Loss Function**

$$L = \sum_{l=1}^k \sum_{i \in C_l} ||x_i - \mu_l||^2 = \sum_{l=1}^k \sum_{i=1}^n a_{il} ||x_i - \mu_l||^2 \to minimize$$

The loss is also called **within-cluster sum of squares** because it sums over all squared distances of samples from the centroid to which they belong. Unknown are the cluster assignments $a_{il}$ and the centroids $\mu_l$.

Algorithmic challenge: given sample vectors $x_i$, determine cluster assignment $a_{il} \in [0,1]$ and centroids $\mu_l$.

#### Expectation-Maximization-like iteration

0. Initialize model: $\mu_l$
1. Iterate until convergence
   1. Assign each $x_i$ to the cluster with nearest centroid: $a_{il}$
   2. Update centroid: $\mu_j = 1/n_j \sum_i x_i$
:::

In scikit-learn, the total within-cluster sum of squares is called `inertia_`. Each cluster has its own within-cluster sum of squares, but often we are interested in the total sum over all clusters: this is what k-means minimizes for a given $k$.

#### Screening cluster solutions

Increasing $k$ will decrease within-cluster SS, but increase model complexity. For $k=n$, each point would be its own centroid and within-cluster SS would be 0. Usually we look for a practical compromise.

#### Cluster Evaluation: Silhouette analysis

The silhouette score measures how well each data point fits within its assigned cluster compared to other clusters. It quantifies **cohesion** and **separation**. The value for each data point ranges from -1 to 1.

A formal definition is given here: [wiki](https://en.wikipedia.org/wiki/Silhouette_(clustering)){target="_blank" rel="noopener"}

### Numerically: Adjusted Rand Index

Different cluster algorithms can produce different partitions and different numbers of clusters. Moreover, the cluster labels are arbitrary. Therefore we need a better method to compare cluster solutions. A common solution is to focus on all *pairs* of samples and consider how often they end up in the same cluster or different clusters, regardless of arbitrary cluster labels.

The Rand Index (Rand 1971) for two partitions is defined as follows:

$$RI = \frac{a+d}{a+b+c+d} \in [0,1]$$

![Same data - two different partitions](figures/RandIndex.jpeg)

The Adjusted Rand Index (ARI) measures the similarity between two cluster solutions, accounting for chance groupings. Specifically, it accounts for an expected RI that would be observed for random cluster assignments and for a maximally possible RI.

It ranges from -1 to 1, with 1 indicating perfect agreement and 0 indicating similarity expected by chance.

$$ARI = \frac{A - E}{M - E} \in [-1, +1]$$

- $A$ = number of cluster agreements of sample pairs
- $M$ = maximal possible number of agreements
- $E$ = expected number of agreements by chance

```{python}
#| label: ARI

ari_score = adjusted_rand_score(cluster_hclust, cluster_kmeans)
print(f"Adjusted Rand Index (ARI): {ari_score:.3f}")
```

## Summary: Cluster Algorithms

Common Goal: **unsupervised** organization and structuring of data

| Hierarchical clustering                     | K-means                                     |
| ------------------------------------------- | ------------------------------------------- |
| bottom-up greedy merging (agglomeration)    | top-down global objective: $L \to min$      |
| No need to choose (K) initially             | Must specify (K)                            |
| Parameters: linkage method, distance        | Parameters: $K$, `n_init`, `max_iter`, `eps`|
| Produces a dendrogram                       | Produces a single partition                 |
| Many linkage methods                        | Simple centroid model                       |
| Deterministic                               | Random initialization                       |
| Expensive: $O(N^2)$                         | CHeap: $O(N)$                               |
| Difficult to update with new data           | Easy to rerun or update                     |


- many more [cluster algorithms](https://en.wikipedia.org/wiki/Category:Cluster_analysis_algorithms){target="_blank" rel="noopener"}

- Metrics:
  - within-cluster SS / inertia $\to 0$
  - silhouette score $\in [-1, 1]$
  - ARI $\in [-1, 1]$
- Expert knowledge $\to$ supervision!
